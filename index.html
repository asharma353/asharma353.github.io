02

Field notes

№ 01 — distributed systems

The joins that wouldn't scale

Terabyte spatial joins kept dying — Sedona's spatial partitioners OOM'd and hung at production scale. I re-architected them as geohash equi-joins, trading a clever-but-fragile abstraction for one that maps to how Spark actually shuffles. The whole class of failures disappeared.

№ 02 — platform

Three forks, one spine

Three divergent forked EMR codebases meant every fix happened three times. I unified them into a shared orchestration layer via a strangler-fig migration — additive, parity-tested across all 12 production configs — with CI/CD from GitHub Actions to ECR replacing manual deploys.
